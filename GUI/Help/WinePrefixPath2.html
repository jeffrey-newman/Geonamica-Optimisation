Wine prefix path

 

This gives the path to the wine prefix to use to run Geonamica. Geonamica itself needs to be installed in this prefix in the path specified in the ‘Geonamica command’ field above.

 

In addition, the way the optimisation uses this prefix is also specified as follows:

 

If field is specified:

“No_wine” or is left blank, then no prefix is specified.

 

“use_home_path” then: the default prefix in “~/.wine” is used.

 

“generate:<path>” then an attempt is made to generate a prefix from scratch within the working directopry. The path to the Geonamica model as given by the <path> and the contents of this folder is copied into the prefix,

 

“copy:<path> then: a copy of the prefix is made in the working directory. This seems to be the best way to go about prefixes, especially when running through mpi.
